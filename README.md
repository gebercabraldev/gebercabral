# 📊 Sistema de Gestão de Recursos

> API robusta em **Spring Boot** para gerenciamento de **funcionários** e **departamentos**, com filtros dinâmicos, ranking por salário, testes automatizados e execução via Docker.

## ✨ Funcionalidades

- Cadastro e edição de funcionários e departamentos
- Filtros dinâmicos por nome, cargo, salário e departamento
- Listagem de funcionários ordenados por **salário**
- Testes com **JUnit 5** e **Mockito**
- Integração com **PostgreSQL** e **Docker Compose**

## 🚀 Como Executar o Projeto

### ✅ Pré-requisitos

- Docker + Docker Compose  
- (Opcional) Java 17+  
- (Opcional) Maven ou Gradle

### 🧱 Passo 1: Gerar o `.jar`

Com Maven:

```bash
./mvnw clean package -DskipTests
```

Com Gradle:

```bash
./gradlew build -x test
```

### 🐳 Passo 2: Subir os containers

```bash
docker-compose up --build
```

- API disponível em: [http://localhost:8080](http://localhost:8080)  
- Banco de Dados (PostgreSQL): porta `5432`

---

## 🧪 Como Rodar os Testes

Execute fora do container:

```bash
./mvnw test         # ou
./gradlew test
```

> Os testes são feitos com **JUnit 5** e **Mockito**.

## 🗄️ Banco de Dados

O projeto usa **PostgreSQL**, configurado automaticamente via Docker Compose.

| Propriedade | Valor         |
|------------:|---------------|
| Host        | `db`          |
| Porta       | `5432`        |
| Banco       | `gestaorecursos` |
| Usuário     | `postgres`    |
| Senha       | `postgres`    |

📌 Pode ser acessado via **pgAdmin**, **DBeaver** ou `psql`.
